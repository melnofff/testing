# Manual Security Testing Report

## Общая информация

**Объект тестирования:** официальный сайт Финансового университета ([https://fa.ru](https://fa.ru))
**Тип тестирования:** ручное тестирование безопасности (Manual Security Testing)
**Цель:** выявление типовых уязвимостей веб‑приложения на уровне пользовательского интерфейса и HTTP‑взаимодействий.

---

## 1. Проверка на XSS (Cross-Site Scripting)

### Описание

Проведена проверка поля поиска и URL‑параметров на наличие отражённых XSS‑уязвимостей.

### Тестовые данные

```html
<script>alert(1)</script>
<img src=x onerror=alert(1)>
```

### Шаги

1. Ввести XSS‑payload в поле поиска.
2. Отправить запрос.
3. Проанализировать отображение результата и поведение браузера.

### Ожидаемый результат

* JavaScript‑код не выполняется.
* Всплывающие окна отсутствуют.
* Ввод либо экранируется, либо не влияет на работу страницы.

### Фактический результат

* Введённые конструкции не выполняются.
* JavaScript не интерпретируется браузером.

### Результат теста

**Уязвимость не обнаружена.**

---

## 2. Проверка на SQL Injection

### Описание

Проведена проверка поля поиска и URL‑параметров на возможность SQL‑инъекций.

### Тестовые данные

```sql
' OR 1=1
' OR '1'='1
' OR 1=1 --
1 OR 1=1
```

### Шаги

1. Ввести SQL‑payload в поле поиска.
2. Отправить запрос.
3. Проанализировать количество и корректность результатов.

### Ожидаемый результат

* Отсутствие SQL‑ошибок.
* Отсутствие сообщений об ошибках базы данных.
* Корректная обработка запроса.

### Фактический результат

* SQL‑ошибки не отображаются.
* Поведение поиска не нарушено.
* Результаты либо отсутствуют, либо соответствуют обычному поиску.

### Результат теста

**Уязвимость не обнаружена.**

---

## 3. Проверка URL‑параметров

### Описание

Проведено тестирование обработки некорректных и вредоносных значений URL‑параметров.

### Тестовые данные

```text
?q=%%%%
?q='
?q=<script>alert(1)</script>
?q=%27%20OR%201=1
```

### Шаги

1. Изменить параметр `q` в URL.
2. Открыть страницу поиска.
3. Проанализировать поведение приложения.

### Ожидаемый результат

* Страница корректно загружается.
* Ошибки сервера отсутствуют.
* Нет раскрытия служебной информации.

### Фактический результат

* Страница загружается корректно.
* Ошибки сервера отсутствуют.

### Результат теста

**Уязвимость не обнаружена.**

---

## 4. Проверка cookies и сессий

### Описание

Проведена проверка cookies на корректность установки и обработки.

### Шаги

1. Открыть сайт.
2. Проанализировать cookies через DevTools (Application → Cookies).
3. Удалить cookies вручную.
4. Перезагрузить страницу.

### Ожидаемый результат

* Cookies пересоздаются корректно.
* Критичные данные не хранятся в cookies в открытом виде.
* Сайт остаётся работоспособным.

### Фактический результат

* Cookies обновляются после перезагрузки страницы.
* Удалённые cookies восстанавливаются автоматически.
* Нарушений в работе сайта не выявлено.

### Результат теста

**Уязвимости не обнаружены.**

---

## 5. Проверка HTTP‑заголовков безопасности

### Описание

Проведён анализ HTTP‑заголовков ответа сервера.

### Проверенные заголовки

* Content-Security-Policy
* X-Frame-Options
* Cache-Control
* Server

### Фактический результат

* Присутствует заголовок `Content-Security-Policy`.
* Присутствует `X-Frame-Options: SAMEORIGIN`.
* Сервер корректно ограничивает встраивание страниц.

### Результат теста

**Базовые заголовки безопасности настроены корректно.**

---

## 6. Проверка обработки ошибок

### Описание

Проверена реакция приложения на некорректный ввод и несуществующие ресурсы.

### Шаги

1. Ввод некорректных данных в поле поиска.
2. Обращение к несуществующим страницам.
3. Анализ отображаемых сообщений.

### Ожидаемый результат

* Отсутствие stack trace.
* Отсутствие SQL/Server ошибок.
* Корректные пользовательские сообщения.

### Фактический результат

* Технические ошибки не отображаются.
* Служебная информация не раскрывается.

### Результат теста

**Уязвимости не обнаружены.**

---

## Общий вывод

В ходе ручного тестирования безопасности критические уязвимости (XSS, SQL Injection, утечки через ошибки, небезопасные cookies) выявлены не были. Приложение корректно обрабатывает пользовательский ввод и ошибки, а также использует базовые механизмы защиты на уровне HTTP‑заголовков.
